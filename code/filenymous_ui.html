Confidentialité par conception

Filenymous n'est pas juste conforme au RGPD. Il est construit de façon à ce qu'il soit structurellement impossible pour un opérateur de lire vos fichiers.

🛡 Les piliers de sécurité du réseau Holochain
🔑
Identité cryptographique
Chaque participant est une paire de clés Ed25519 générée localement. Aucune autorité centrale ne délivre d'identité. Vos actions sur le réseau sont signées par votre clé privée, qui ne quitte jamais votre appareil.
🧩
Fragmentation chiffrée
Les fichiers sont découpés en chunks de 256 Ko, chiffrés individuellement avec AES-256-GCM, puis distribués sur des nœuds différents du DHT. Aucun nœud ne possède un fichier complet, ni la clé pour le déchiffrer.
🌐
DHT sans coordinateur
La Distributed Hash Table Holochain stocke les données de façon redondante sans serveur maître. Il n'existe pas de point unique d'attaque, de saisie, ou de censure. Couper un nœud ne compromet pas les données.
📜
Source chain immuable
Chaque agent tient un journal local signé et chaîné de toutes ses actions (source chain). La falsification d'une action est détectable par l'ensemble du réseau, sans blockchain ni consensus global coûteux.
⏱
Expiration garantie
L'expiration d'un transfert publie une DeleteAction sur le DHT. Les nœuds stockant les chunks sont obligés par la DNA (contrat applicatif) de supprimer les données. Ce n'est pas une promesse — c'est une règle cryptographiquement validée.
🕵️
Identité des contacts minimale
Votre email ou téléphone n'est jamais publié sur le réseau. Seul son hash SHA-256 est stocké sur le DHT, associé à votre clé publique. Un observateur externe ne peut pas reconstituer votre adresse depuis ce hash.
📋 Mentions légales & RGPD

Responsable de traitement

Filenymous est un protocole décentralisé open-source. En l'absence de base de données centrale, il n'existe pas de "responsable de traitement" au sens classique du RGPD pour les données des fichiers transférés. Chaque utilisateur est responsable des données qu'il publie sur le DHT.

Données collectées

Le bridge de notification (envoi email/SMS) reçoit temporairement l'adresse de destination et le lien de transfert. Ces données ne sont pas persistées au-delà de l'envoi du message (TTL : 0). Aucun cookie de traçage, aucune analytics, aucune publicité.

Droit à l'effacement

L'expéditeur peut révoquer un transfert à tout moment depuis l'onglet Historique. La révocation publie une suppression cryptographique sur le DHT. Les chunks chiffrés sont effacés par les nœuds participants dans un délai de 24h maximum.

Hébergement des données

Les chunks chiffrés sont distribués sur les nœuds actifs du réseau Holochain Filenymous. Aucun chunk n'est localisé sur un serveur géré par l'éditeur. La localisation géographique des nœuds est inconnue par conception (réseau pair-à-pair décentralisé).
