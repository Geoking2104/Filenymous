    Holochain security pillars

    Key

    Cryptographic identity

    Each participant is a locally generated Ed25519 key pair. No central authority issues identity, and your private key never leaves your device.

    Chunks

    Encrypted fragmentation

    Files are split into 512 KB chunks, encrypted with AES-256-GCM, then distributed across different DHT nodes. No node holds a complete file.

    DHT

    Coordinator-free DHT

    The Holochain DHT stores data redundantly without a master server. There is no single attack, seizure, or censorship point.

    TTL

    Enforced expiry

    Expiry publishes a DeleteAction on the DHT. Participating nodes are required by DNA validation rules to remove the data.

    Hash

    Minimal contacts

    Your email or phone number is never published on the network. Only its SHA-256 hash is stored with your public key.

    Log

    Immutable source chain

    Each agent keeps a signed local chain. Forging an action is detectable by the network without blockchain consensus.

    Legal and GDPR notes

    Data controller

    Filenymous is a decentralized open-source protocol. Without a central file database, there is no classic data controller for transferred files. Each user is responsible for the data they publish to the DHT.

    Collected data

    The notification bridge may temporarily receive the destination address and transfer link for email/SMS delivery. This data is not persisted beyond sending. There are no tracking cookies, analytics, or advertising.

    Right to erasure

    The sender can revoke a transfer from the History tab. Revocation publishes a cryptographic delete action on the DHT, and participating nodes remove chunks under the app rules.

    Data hosting

    Encrypted chunks are distributed across active Holochain Filenymous network nodes. No chunk is located on an editor-managed file server, and node geography is unknown by design.
